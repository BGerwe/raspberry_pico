  16. /*
  17.  * Here, we'll list a few important ideas and parts that we'll learn about today:
  18.  * - A Circuit: It's like a circle that electricity travels around. It needs to be closed, with no breaks.
  19.  * - Current: It's the flow of electricity.
  20.  * - Voltage: It's the force pushing the electricity.
  21.  * - Resistance: It's something that slows down the electricity.
  22.  *
  23.  * And here are some things we'll use with our Arduino HERO:
  24.  * - Breadboard: A tool that helps us build our circuit without any soldering.
  25.  * - 5V pins: These are the pins on our HERO that give 5 volts of power.

  28.  * We'll also learn about these programming ideas:

  32.  * - Functions: These are chunks of code that perform a specific task.

  35. // We start by including this line of code, which helps our HERO work properly with the Arduino program.
  36. #include "Arduino.h"
  37.  
  38. /*
  39.  * Today we are going to make a big light turn on to light up our lander! We'll use an LED for now to
  40.  * test our setup. We will use a digital pin on our HERO to control our lander's light.
  41.  *
  42.  * In our code, we often use constants. These are like nicknames that we give to specific values. This
  43.  * makes our code easier to read and change later if needed. For example, instead of remembering a pin
  44.  * number, we can just give it a nickname like CABIN_LIGHTS_PIN.
  45.  *
  46.  * One way to define constants is by using a command called #define.  Using this we can give a name
  47.  * (or nickname) to the value that follows the name. You'll notice that the nickname is in capital letters,
  48.  * which is a way to remind us that it's a constant and can't change.
  49.  *
  50.  * In this #define we give the name "CABIN_LIGHTS_PIN" the value of "12".  Whenever you see
  51.  * in this sketch it will have the value of "12".
  52.  */
  53. #define CABIN_LIGHTS_PIN 12   // This is the nickname for the pin connected to our LED.
  54.  
  55. /*
  56.  * The setup function is where we prepare everything before we start. We only run this function once each
  57.  * time we turn on our HERO or reset it. Here, we are setting up our CABIN_LIGHTS_PIN as an output pin, so
  58.  * we can control our light with it.
  59.  */
  60. void setup() {
  61.   pinMode(CABIN_LIGHTS_PIN, OUTPUT);    // We will control our lander's lights as an OUTPUT.
  62. }
  63.  
  64. /*
  65.  * After the setup function, the loop function starts. This function keeps repeating over and over, like
  66.  * a circle. Here, we are making our light turn on and off in a repeating pattern.
  67.  */
  68. void loop() {
  69.   digitalWrite(CABIN_LIGHTS_PIN, HIGH);  // This line turns the lander's light ON.
  70.   delay(1000);                           // Wait for one second (1000 milliseconds) with the light ON.
  71.   digitalWrite(CABIN_LIGHTS_PIN, LOW);   // This line turns the lander's light OFF.
  72.   delay(100);                            // Wait for a tenth of a second (100 milliseconds) with the light OFF.
  73. }
